# 部署说明

本文档记录 `liankao.index545.com` 的生产部署和后续更新流程。

## 服务器信息

- 登录方式：`ssh liankao`
- 代码目录：`/opt/liankao-ng/app`
- 生产环境变量：`/etc/liankao-ng/server.env`
- SQLite 数据库：`/var/lib/liankao-ng/data.db`
- 小视野 fetcher：通过 `XSY_FETCHER_URL` 和 `XSY_FETCHER_TOKEN` 配置
- 后端服务：`liankao-server`
- 后端监听：`127.0.0.1:3000`
- Nginx 站点配置：`/etc/nginx/sites-available/liankao`
- 访问地址：`https://liankao.index545.com/`
- API 文档：`https://liankao.index545.com/api/openapi`

证书由 `acme.sh` 管理，当前 Nginx 使用：

```text
/home/ubuntu/.acme.sh/index545.com_ecc/fullchain.cer
/home/ubuntu/.acme.sh/index545.com_ecc/index545.com.key
```

## 常规更新部署

本地提交并 push 到 GitHub 后，在服务器执行：

```bash
ssh liankao

cd /opt/liankao-ng/app
git fetch origin
git checkout origin/main
git reset --hard origin/main

bun install --frozen-lockfile

cd /opt/liankao-ng/app/packages/server
set -a
. /etc/liankao-ng/server.env
set +a
bunx --bun prisma generate

cd /opt/liankao-ng/app/packages/client
bun run build

sudo systemctl restart liankao-server
sudo nginx -t
sudo systemctl reload nginx
```

## 数据库注意事项

生产 SQLite 数据库在仓库目录之外：

```text
/var/lib/liankao-ng/data.db
```

常规更新不要删除、覆盖或从本地同步这个文件。

## 小视野 Fetcher

如果生产服务器无法直接访问小视野，在腾讯云 SCF 部署国内 fetcher：

```bash
cd packages/xsy-fetcher
scf deploy
```

部署时需要在腾讯云函数环境变量里设置：

```env
XSY_FETCHER_TOKEN=replace-with-a-long-random-secret
XSY_REQUEST_TIMEOUT_MS=8000
```

拿到 API 网关地址后，在主后端生产环境变量 `/etc/liankao-ng/server.env` 中加入：

```env
XSY_FETCHER_URL=https://service-xxx.gz.apigw.tencentcs.com/release
XSY_FETCHER_TOKEN=replace-with-the-same-secret
XSY_FETCHER_TIMEOUT_MS=10000
```

然后重启后端：

```bash
sudo systemctl restart liankao-server
```

未配置 `XSY_FETCHER_URL` 时，后端会保持原来的直连小视野行为。

如果 Prisma schema 发生变化，先备份数据库：

```bash
sudo cp -a /var/lib/liankao-ng/data.db /var/lib/liankao-ng/data.db.backup.$(date +%Y%m%d%H%M%S)
```

当前项目没有 `prisma/migrations` 目录。需要把 schema 变更应用到生产库时，在确认变更安全后执行：

```bash
cd /opt/liankao-ng/app/packages/server
set -a
. /etc/liankao-ng/server.env
set +a
bunx --bun prisma db push
sudo systemctl restart liankao-server
```

后续如果项目引入 Prisma migrations，生产迁移应改用：

```bash
bunx --bun prisma migrate deploy
```

## 服务检查

```bash
sudo systemctl status liankao-server --no-pager
sudo systemctl status nginx --no-pager

curl -fsSI http://127.0.0.1:3000/api/openapi
curl -fsSI --resolve liankao.index545.com:443:127.0.0.1 https://liankao.index545.com/
curl -fsSI --resolve liankao.index545.com:443:127.0.0.1 https://liankao.index545.com/api/openapi
```

DNS 生效后可直接检查：

```bash
curl -fsSI https://liankao.index545.com/
curl -fsSI https://liankao.index545.com/api/openapi
```

查看日志：

```bash
journalctl -u liankao-server -n 100 --no-pager
sudo tail -n 100 /var/log/nginx/access.log
sudo tail -n 100 /var/log/nginx/error.log
```

## 回滚

回滚到上一个可用提交：

```bash
cd /opt/liankao-ng/app
git fetch origin
git checkout <commit-sha>

bun install --frozen-lockfile

cd /opt/liankao-ng/app/packages/server
set -a
. /etc/liankao-ng/server.env
set +a
bunx --bun prisma generate

cd /opt/liankao-ng/app/packages/client
bun run build

sudo systemctl restart liankao-server
sudo nginx -t
sudo systemctl reload nginx
```

如果回滚涉及数据库 schema 或数据变化，先确认数据库备份和恢复方案，不要直接覆盖生产数据库。
